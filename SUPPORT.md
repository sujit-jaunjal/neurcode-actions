# Support

## Where to get help

- Bugs and feature requests: GitHub Issues
- Product and usage questions: GitHub Discussions
- Security concerns: follow [SECURITY.md](SECURITY.md)

## What to include in bug reports

- the action version / tag you pinned (e.g. `@v0.2.4`)
- your workflow snippet (the `with:` block)
- the `scope_hash` from the run (so the result can be reproduced)
- expected vs. actual behavior
- the run's Step Summary and, if relevant, the posted comment
- repo language/build system (helps reproduce topology)
